## 2. Shared Types

All type definitions used across modules.

### 2.1 `Cue` TypedDict

```python
class Cue(TypedDict):
    cue_id: int              # Sequential index (0-based) after parsing
    start_seconds: float     # Timestamp converted to total seconds
    end_seconds: float
    start_timestamp: str     # Original "HH:MM:SS.mmm"
    end_timestamp: str
    text: str                # Cleaned text (speaker labels stripped, whitespace normalized)
```

### 2.2 `MergedCue` TypedDict

Extends `Cue` with merge metadata.

```python
class MergedCue(TypedDict):
    cue_id: int              # cue_id of the first cue in the merge group
    start_seconds: float
    end_seconds: float
    start_timestamp: str
    end_timestamp: str
    text: str                # Concatenated text from merged cues
    merged_count: int        # Number of original cues merged into this one
    potential_boundary: bool # Set by topic_detector, defaults to False
```

### 2.3 `ChunkCue` TypedDict

A cue as it appears inside a chunk (after overlap is applied).

```python
class ChunkCue(TypedDict):
    cue_id: int
    start_seconds: float
    end_seconds: float
    start_timestamp: str
    end_timestamp: str
    text: str
    merged_count: int
    potential_boundary: bool
    is_overlap: bool         # True if prepended from previous chunk for overlap
```

### 2.4 `Chunk` TypedDict

Final chunk object output by the pipeline.

```python
class Chunk(TypedDict):
    chunk_id: int            # Sequential (0-indexed)
    text: str                # Full joined text of all cues in chunk
    start_time: str          # "HH:MM:SS" of first cue
    end_time: str            # "HH:MM:SS" of last cue
    start_seconds: float
    end_seconds: float
    duration_seconds: float
    cue_count: int           # Number of merged cues (excluding overlap cues)
    topic_hint: str          # First sentence (up to first period or 120 chars)
    chunk_type: str          # "code_walkthrough" | "qa" | "explanation"
    source_file: str         # Original .vtt filename
    cues: list[ChunkCue]    # All cues in this chunk (including overlap)
```

### 2.5 `ChunkSummary` TypedDict

Lightweight summary for `chunks_summary.json`.

```python
class ChunkSummary(TypedDict):
    chunk_id: int
    start_time: str
    end_time: str
    topic_hint: str
    chunk_type: str
    cue_count: int
```

### 3.3 `vtt_parser.py` — Phase 1

**Purpose**: Parse a raw `.vtt` file into a list of `Cue` objects.

#### 3.3.1 `parse_vtt`

```python
def parse_vtt(file_path: str) -> list[Cue]:
    """Parse a VTT file into a list of Cue objects.

    Args:
        file_path: Absolute or relative path to a .vtt file.

    Returns:
        List of Cue dicts, ordered by start_seconds ascending.

    Raises:
        FileNotFoundError: If file_path does not exist.
        ValueError: If the file is not a valid VTT (missing WEBVTT header).
    """
```

**Algorithm**:

1. Read file as text with `encoding="utf-8"`
2. Normalize line endings: replace `\r\n` → `\n`
3. Verify first line starts with `WEBVTT` (case-sensitive, may have trailing info like `WEBVTT - vtt`). If not → raise `ValueError`
4. Split into blocks separated by one or more blank lines
5. For each block:
   a. Skip if block is empty or only contains the WEBVTT header
   b. Iterate lines within block:
      - If line matches `^\d+$` (standalone integer) → skip (cue index)
      - If line matches `^\d{2}:\d{2}:\d{2}\.\d{3}\s*-->\s*\d{2}:\d{2}:\d{2}\.\d{3}` → extract start/end timestamps
      - Otherwise → collect as text content lines
   c. If block has both timestamps and text → create a Cue
   d. Clean the text:
      - Join multiple text lines with a single space
      - Strip speaker labels: match pattern `^[A-Z]+:\s*` at the start of the text and remove it (e.g. `"PAUL: Hello"` → `"Hello"`)
      - Strip `<i>`, `</i>`, `<b>`, `</b>`, `<u>`, `</u>` HTML-like tags
      - Strip leading/trailing whitespace
      - Collapse multiple spaces into one
   e. Skip cue if cleaned text is empty
6. Assign sequential `cue_id` starting from 0

#### 3.3.2 `_parse_timestamp`

```python
def _parse_timestamp(ts: str) -> float:
    """Convert a VTT timestamp string to total seconds.

    Args:
        ts: Timestamp in "HH:MM:SS.mmm" format.

    Returns:
        Total seconds as float.

    Example:
        "01:23:45.678" → 5025.678
    """
```

**Algorithm**:
1. Split on `:` → hours, minutes, rest
2. Split rest on `.` → seconds, milliseconds
3. Compute: `hours * 3600 + minutes * 60 + seconds + milliseconds / 1000`

#### 3.3.3 `_format_timestamp`

```python
def _format_timestamp(total_seconds: float) -> str:
    """Convert total seconds to "HH:MM:SS" format string.

    Args:
        total_seconds: Seconds as float.

    Returns:
        Timestamp string like "01:23:45" (no milliseconds).

    Example:
        5025.678 → "01:23:45"
    """
```

**Edge cases**:
- File with no cues → return empty list (no error)
- Cues out of order → preserve original order, do not re-sort
- Malformed timestamp line → skip that block, log warning
- Duplicate cue indices → ignore indices entirely, assign our own sequential IDs
- UTF-8 BOM at start of file → strip it

### 3.4 `cue_merger.py` — Phase 2

**Purpose**: Merge consecutive short cues into sentence-level cues.

#### 3.4.1 `merge_cues`

```python
def merge_cues(cues: list[Cue]) -> list[MergedCue]:
    """Merge consecutive short cues into sentence-level merged cues.

    Args:
        cues: List of Cue objects from Phase 1.

    Returns:
        List of MergedCue objects with potential_boundary set to False.
    """
```

**Algorithm** (single left-to-right pass):

1. Initialize `result: list[MergedCue] = []`
2. Initialize `buffer_text = cues[0].text`, `buffer_start = cues[0]`, `buffer_count = 1`, `buffer_end_seconds = cues[0].end_seconds`, `buffer_end_timestamp = cues[0].end_timestamp`
3. For each subsequent cue `cues[i]` (i from 1 to len-1):
   a. Compute `gap = cue.start_seconds - buffer_end_seconds`
   b. **Merge check — do NOT merge if any of these are true**:
      - `gap > MERGE_GAP_THRESHOLD_SECONDS` → gap too large
      - `len(buffer_text) + 1 + len(cue.text) > MERGE_MAX_CHARS` → would exceed char limit
      - `buffer_text.rstrip().endswith(('.', '?', '!'))` AND `cue.text.lstrip()[0:1].isupper()` → sentence boundary detected (current cue ends with sentence terminator, next starts with capital)
   c. If any no-merge condition is true → **flush buffer**:
      - Create `MergedCue` from buffer state, append to `result`
      - Reset buffer with current cue
   d. Otherwise → **merge**:
      - Append `cue.text` to `buffer_text` with a space separator
      - Update `buffer_end_seconds = cue.end_seconds`
      - Update `buffer_end_timestamp = cue.end_timestamp`
      - Increment `buffer_count`
4. After loop → flush remaining buffer as final `MergedCue`
5. Set `potential_boundary = False` on all merged cues
6. Return `result`

**Edge cases**:
- Empty input list → return empty list
- Single cue → return single MergedCue with `merged_count=1`
- All cues are long / large gaps → no merging occurs, each cue becomes its own MergedCue
- Cue text starting with lowercase after sentence end → still merges (only capital-letter check triggers boundary)

### 3.5 `topic_detector.py` — Phase 3

**Purpose**: Mark merged cues as potential topic boundaries.

#### 3.5.1 `detect_boundaries`

```python
def detect_boundaries(merged_cues: list[MergedCue]) -> list[MergedCue]:
    """Detect topic boundaries and set potential_boundary on merged cues.

    Args:
        merged_cues: List of MergedCue objects from Phase 2.

    Returns:
        Same list with potential_boundary field updated in-place.
        (Also returned for chaining convenience.)
    """
```

**Algorithm** (two independent checks per cue):

1. For each `merged_cues[i]`:
   a. **Keyword check**: Convert `text` to lowercase. Check if any phrase in `TOPIC_SIGNAL_PHRASES` is present as a substring. If yes → set `potential_boundary = True`
   b. **Silence gap check**: If `i > 0`, compute `gap = merged_cues[i].start_seconds - merged_cues[i-1].end_seconds`. If `gap > SILENCE_GAP_BOUNDARY_SECONDS` → set `potential_boundary = True`
2. Return the same list (mutated in-place)

#### 3.5.2 `_contains_topic_signal`

```python
def _contains_topic_signal(text: str) -> bool:
    """Check if text contains any topic-signal phrase.

    Args:
        text: Cue text to check.

    Returns:
        True if any trigger phrase is found (case-insensitive).
    """
```

**Algorithm**:
1. Convert `text` to lowercase
2. Iterate `TOPIC_SIGNAL_PHRASES`
3. If any phrase is found as substring → return True
4. Otherwise → return False

**Note**: Matching is simple substring, not word-boundary. This is intentional — `"step "` has a trailing space to avoid matching `"steps"` but match `"step 1"`. Phrases like `"next,"` include the comma to match conversational use.

**Edge cases**:
- Empty text → no match, boundary stays False
- Multiple trigger phrases in one cue → still just `potential_boundary = True` (boolean, not scored)
- First cue (i=0) → no silence gap check possible, only keyword check applies

### 3.6 `chunk_builder.py` — Phase 4

**Purpose**: Group merged cues into chunks with overlap between them.

#### 3.6.1 `build_chunks`

```python
def build_chunks(merged_cues: list[MergedCue]) -> list[list[ChunkCue]]:
    """Build chunks from merged cues with topic-boundary splitting and overlap.

    Args:
        merged_cues: List of MergedCue objects with potential_boundary set.

    Returns:
        List of chunks, where each chunk is a list of ChunkCue objects.
    """
```

**Algorithm — Step 4.1 (chunk creation)**:

1. Initialize `chunks: list[list[ChunkCue]] = []`
2. Initialize `buffer: list[ChunkCue] = []`
3. For each `merged_cue` in `merged_cues`:
   a. Convert `MergedCue` → `ChunkCue` (copy all fields, set `is_overlap = False`)
   b. Append to `buffer`
   c. **Flush check** (apply in order):
      - If `merged_cue.potential_boundary == True` AND `len(buffer) >= TOPIC_SIGNAL_MIN_CUES` → flush buffer as a chunk, reset buffer to empty
      - Else if `len(buffer) >= CHUNK_HARD_CAP_CUES` → flush buffer as a chunk, reset buffer to empty
4. After loop → if `buffer` is non-empty, flush as final chunk
5. Apply overlap (Step 4.2)

**Algorithm — Step 4.2 (add overlap)**:

1. For each pair of adjacent chunks `chunks[i]` and `chunks[i+1]` (i from 0 to len-2):
   a. Take the last `OVERLAP_CUES` cues from `chunks[i]`
   b. If `chunks[i]` has fewer than `OVERLAP_CUES` cues → take all available
   c. Create copies of those cues with `is_overlap = True`
   d. Prepend those copies to the start of `chunks[i+1]`
2. Return `chunks`

**Edge cases**:
- Empty input → return empty list
- Single cue → return single chunk with one cue
- All cues have `potential_boundary = True` → chunks will be `TOPIC_SIGNAL_MIN_CUES` cues each
- No boundaries detected → one large chunk (or chunks split by hard cap only)
- Chunk with fewer cues than `OVERLAP_CUES` → overlap takes whatever is available

### 3.7 `metadata_enricher.py` — Phase 5

**Purpose**: Compute per-chunk metadata and auto-tag chunk_type.

#### 3.7.1 `enrich`

```python
def enrich(
    chunks: list[list[ChunkCue]],
    source_file: str,
) -> list[Chunk]:
    """Enrich chunks with metadata and chunk_type classification.

    Args:
        chunks: List of chunks (each a list of ChunkCue objects) from Phase 4.
        source_file: Original .vtt filename for metadata.

    Returns:
        List of Chunk dicts with full metadata.
    """
```

**Algorithm**:

For each chunk at index `i`:

1. Filter out overlap cues for `text` and `cue_count`:
   - `non_overlap_cues = [c for c in chunk if not c["is_overlap"]]`
2. Compute `text`: join all non-overlap cue texts with space separator
3. Compute `start_time`: format `non_overlap_cues[0].start_seconds` as `HH:MM:SS`
4. Compute `end_time`: format `non_overlap_cues[-1].end_seconds` as `HH:MM:SS`
5. Compute `start_seconds`, `end_seconds` from first/last non-overlap cues
6. Compute `duration_seconds = end_seconds - start_seconds`
7. Compute `cue_count = len(non_overlap_cues)`
8. Compute `topic_hint`:
   - Take first sentence of `text` (split on `.`, `?`, `!`)
   - If no sentence terminator found → take first `TOPIC_HINT_MAX_CHARS` characters
   - If sentence exceeds `TOPIC_HINT_MAX_CHARS` → truncate to `TOPIC_HINT_MAX_CHARS` with `...`
9. Classify `chunk_type` via `_classify_chunk_type(text)`
10. Build `Chunk` dict with all fields + the full `cues` list (including overlap cues)

#### 3.7.2 `_classify_chunk_type`

```python
def _classify_chunk_type(text: str) -> str:
    """Classify a chunk's text as code_walkthrough, qa, or explanation.

    Args:
        text: Full chunk text.

    Returns:
        One of: "code_walkthrough", "qa", "explanation".
    """
```

**Algorithm**:

1. Convert `text` to lowercase
2. **Check code_walkthrough first** (higher priority):
   - If any keyword in `CODE_WALKTHROUGH_KEYWORDS` is found as substring → return `"code_walkthrough"`
3. **Then check qa**:
   - If any keyword in `QA_KEYWORDS` is found as substring → return `"qa"`
4. **Default** → return `"explanation"`

**Priority rule**: `code_walkthrough` > `qa` > `explanation`. If both code and QA keywords are present, code wins. This is intentional — a Q&A about code should be classified as code_walkthrough.

#### 3.7.3 `_extract_topic_hint`

```python
def _extract_topic_hint(text: str) -> str:
    """Extract the first sentence of text as a topic hint.

    Args:
        text: Full chunk text.

    Returns:
        First sentence, truncated to TOPIC_HINT_MAX_CHARS if needed.
    """
```

**Algorithm**:

1. For each of `.`, `?`, `!`:
   - Find the index of the first occurrence
2. Take the minimum positive index among those → `sentence_end`
3. If no sentence terminator found → `sentence_end = min(len(text), TOPIC_HINT_MAX_CHARS)`
4. `hint = text[:sentence_end+1]` (include the terminator)
5. If `len(hint) > TOPIC_HINT_MAX_CHARS` → truncate to `TOPIC_HINT_MAX_CHARS - 3` and append `"..."`
6. Strip whitespace and return

### 3.8 `pipeline.py` — Phase 6

**Purpose**: Orchestrate all phases, write output, validate.

#### 3.8.1 `run_vtt_chunking_pipeline`

```python
def run_vtt_chunking_pipeline(
    vtt_file_path: str,
    output_dir: str = "data/chunks",
) -> dict:
    """Run the full VTT chunking pipeline.

    Args:
        vtt_file_path: Path to the .vtt file.
        output_dir: Directory for output JSON files.

    Returns:
        dict with keys:
            "chunks": list[Chunk]    — full chunk objects
            "summary": list[ChunkSummary] — summary list
            "validation": dict       — validation results

    Raises:
        FileNotFoundError: If vtt_file_path does not exist.
        ValueError: If VTT parsing fails or validation assertions fail.
    """
```

**Algorithm**:

1. **Phase 1**: `cues = parse_vtt(vtt_file_path)`
2. **Phase 2**: `merged = merge_cues(cues)`
3. **Phase 3**: `merged = detect_boundaries(merged)`
4. **Phase 4**: `raw_chunks = build_chunks(merged)`
5. **Phase 5**: `source_file = os.path.basename(vtt_file_path)`, `chunks = enrich(raw_chunks, source_file)`
6. **Phase 6**:
   a. Create `output_dir` if it doesn't exist (`os.makedirs(output_dir, exist_ok=True)`)
   b. Build `summary = [_chunk_to_summary(c) for c in chunks]`
   c. Write `chunks.json`: `json.dump(chunks, output_dir / "chunks.json", indent=2)`
   d. Write `chunks_summary.json`: `json.dump(summary, output_dir / "chunks_summary.json", indent=2)`
   e. Run validation: `validation = _validate_chunks(chunks)`
   f. Print summary to stdout
7. Return `{"chunks": chunks, "summary": summary, "validation": validation}`

#### 3.8.2 `_chunk_to_summary`

```python
def _chunk_to_summary(chunk: Chunk) -> ChunkSummary:
    """Extract summary fields from a full chunk object.

    Args:
        chunk: Full Chunk dict.

    Returns:
        ChunkSummary with chunk_id, start_time, end_time, topic_hint, chunk_type, cue_count.
    """
```

#### 3.8.3 `_validate_chunks`

```python
def _validate_chunks(chunks: list[Chunk]) -> dict:
    """Validate chunk output against quality thresholds.

    Args:
        chunks: List of Chunk dicts.

    Returns:
        dict with keys:
            "passed": bool
            "total_chunks": int
            "avg_duration_seconds": float
            "type_distribution": dict[str, int]
            "warnings": list[str]
            "errors": list[str]
    """
```

**Validation logic**:

1. `total_chunks = len(chunks)`
2. `avg_duration = mean(c["duration_seconds"] for c in chunks)`
3. `type_distribution = Counter(c["chunk_type"] for c in chunks)`
4. Initialize `errors: list[str] = []`, `warnings: list[str] = []`
5. **Errors** (pipeline-breaking):
   - If `total_chunks < MIN_TOTAL_CHUNKS` or `total_chunks > MAX_TOTAL_CHUNKS` → error: `"Total chunks {total_chunks} outside expected range [{MIN_TOTAL_CHUNKS}-{MAX_TOTAL_CHUNKS}]"`
   - If any chunk has `len(text) < MIN_CHUNK_TEXT_LENGTH` → error: `"Chunk {chunk_id} has text under {MIN_CHUNK_TEXT_LENGTH} chars"`
   - If any chunk has `len(text) > MAX_CHUNK_TEXT_LENGTH` → error: `"Chunk {chunk_id} has text over {MAX_CHUNK_TEXT_LENGTH} chars"`
6. **Warnings** (non-breaking):
   - If any chunk has `cue_count < 3` → warning: `"Chunk {chunk_id} has only {cue_count} cues"`
   - If `type_distribution` has no `code_walkthrough` → warning: `"No code_walkthrough chunks detected"`
   - If `type_distribution` has no `qa` → warning: `"No qa chunks detected"`
7. `passed = len(errors) == 0`
8. Return validation dict

## 5. Error Handling Strategy

### 5.1 Error Types

| Error Type | Module | Handling |
|---|---|---|
| File not found | `vtt_parser` | Raise `FileNotFoundError` |
| Invalid VTT format | `vtt_parser` | Raise `ValueError` with descriptive message |
| Malformed timestamp | `vtt_parser` | Skip block, log warning, continue |
| Empty VTT (no cues) | `vtt_parser` | Return empty list (not an error) |
| Empty cue after cleaning | `vtt_parser` | Skip cue, continue |
| Validation failure | `pipeline` | Return validation dict with `passed=False` and errors list; do NOT raise exception |
| Output dir creation failure | `pipeline` | Raise `OSError` |
| JSON write failure | `pipeline` | Raise `OSError` |

### 5.2 Logging

- Use `structlog` (already in `requirements.txt`) via `src.logger.custom_logger`
- Log at `INFO` level: pipeline start/end, chunk counts per phase
- Log at `WARNING` level: skipped blocks, validation warnings
- Log at `ERROR` level: validation errors
- No logging inside merge/loop internals (too noisy) — only at phase boundaries
