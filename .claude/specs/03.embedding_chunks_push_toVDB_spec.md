## 2. Shared Types

### 2.1 `ChunkRecord` TypedDict

Input record from `chunks.json`.

```python
class ChunkRecord(TypedDict):
    chunk_id: int
    text: str
    start_time: str
    end_time: str
    start_seconds: float
    end_seconds: float
    duration_seconds: float
    cue_count: int
    topic_hint: str
    chunk_type: str  # "code_walkthrough" | "qa" | "explanation"
    source_file: str
    cues: list[dict]  # Full cue objects
```

### 2.2 `EmbeddedChunk` TypedDict

Chunk with embedding vector.

```python
class EmbeddedChunk(TypedDict):
    chunk_id: int
    text: str
    embedding: list[float]  # 1536-dim vector from text-embedding-3-small
    metadata: ChunkMetadata
```

### 2.3 `ChunkMetadata` TypedDict

Metadata stored in vector DB.

```python
class ChunkMetadata(TypedDict):
    chunk_id: int
    start_time: str
    end_time: str
    start_seconds: float
    end_seconds: float
    duration_seconds: float
    topic_hint: str
    chunk_type: str
    source_file: str
```

Status tracking for each chunk.

```python
class EmbeddingStatus(TypedDict):
```

## 3. File Specifications

### 3.1 `chunk_reader.py`

**Purpose**: Read and validate `chunks.json` from chunking pipeline.

#### 3.1.1 `read_chunks`

```python
def read_chunks(chunks_json_path: str) -> list[ChunkRecord]:
    """Read and validate chunks.json output from chunking pipeline.

    Args:
        chunks_json_path: Path to chunks.json file.

    Returns:
        List of ChunkRecord dicts, sorted by chunk_id.

    Raises:
        FileNotFoundError: If chunks_json_path does not exist.
        ValueError: If JSON is malformed or missing required fields.
    """
```

**Algorithm**:

1. Check file exists → raise `FileNotFoundError` if not
2. Open and load JSON with `json.load()`
3. Validate top-level is a list
4. For each item, validate required fields: `chunk_id`, `text`, `chunk_type`, `source_file`
5. Sort list by `chunk_id` ascending
6. Return sorted list

### 3.2 `embedding_client.py`

**Purpose**: Generate embeddings using MeshAPI (OpenAI-compatible).

#### 3.2.1 `EmbeddingClient` Class

```python
class EmbeddingClient:
    """Client for generating text embeddings via MeshAPI."""

    def __init__(self, api_key: str | None = None, base_url: str = "https://api.meshapi.ai/v1"):
        """Initialize the embedding client.

        Args:
            api_key: MeshAPI key. Defaults to MESH_API_KEY env var.
            base_url: MeshAPI base URL.

        Raises:
            ValueError: If no API key provided or found in environment.
        """
```

**Algorithm**:

1. If `api_key` is None → read from `MESH_API_KEY` environment variable
2. If still None → raise `ValueError("MESH_API_KEY not provided")`
3. Store `api_key` and `base_url` as instance attributes
4. Initialize `openai.OpenAI` client with these credentials

#### 3.2.2 `embed_text`

```python
def embed_text(self, text: str) -> list[float]:
    """Generate embedding for a single text.

    Args:
        text: Input text to embed.

    Returns:
        1536-dimensional embedding vector.

    Raises:
        ValueError: If text is empty or too long (>8000 tokens).
        RuntimeError: If API call fails.
    """
```

**Algorithm**:

1. Validate text is non-empty → raise `ValueError` if empty
2. Truncate/validate length (MeshAPI will error if too long)
3. Call `self.client.embeddings.create()` with:
   - `model="openai/text-embedding-3-small"`
   - `input=text`
4. Extract `response.data[0].embedding` (list of floats)
5. Return embedding vector

#### 3.2.3 `embed_batch`

```python
def embed_batch(self, texts: list[str], batch_size: int = 100) -> list[list[float]]:
    """Generate embeddings for multiple texts in batches.

    Args:
        texts: List of input texts to embed.
        batch_size: Number of texts per API call.

    Returns:
        List of embedding vectors in same order as input.

    Raises:
        RuntimeError: If any API call fails.
    """
```

**Algorithm**:

1. Initialize `results: list[list[float]] = []`
2. For `i` from 0 to `len(texts)` step `batch_size`:
   a. `batch = texts[i:i+batch_size]`
   b. Call `self.client.embeddings.create()` with batch input
   c. Extract embeddings in order
   d. Append to `results`
3. Return `results`

### 3.3 `vector_store_client.py`

**Purpose**: Store and query embeddings in AstraDB.

#### 3.3.1 `VectorStoreClient` Class

```python
class VectorStoreClient:
    """Client for AstraDB vector store operations."""

    def __init__(
        self,
        api_endpoint: str | None = None,
        token: str | None = None,
        collection_name: str = "class_chunks",
    ):
        """Initialize AstraDB vector store client.

        Args:
            api_endpoint: AstraDB API endpoint. Defaults to ASTRA_API_ENDPOINT env var.
            token: AstraDB token. Defaults to ASTRA_TOKEN env var.
            collection_name: Name of the collection to use/create.

        Raises:
            ValueError: If credentials not provided or found in environment.
        """
```

**Algorithm**:

1. If `api_endpoint` is None → read from `ASTRA_API_ENDPOINT` env var
2. If `token` is None → read from `ASTRA_TOKEN` env var
3. If either is None → raise `ValueError`
4. Store `collection_name` as instance attribute
5. Initialize `DataAPIClient()` and call `get_database(api_endpoint, token)`
6. Store database reference

#### 3.3.2 `ensure_collection`

```python
def ensure_collection(self, dimension: int = 1536) -> Collection:
    """Ensure the collection exists with correct vector dimensions.

    Args:
        dimension: Vector dimension (1536 for text-embedding-3-small).

    Returns:
        AstraDB Collection object.

    Raises:
        RuntimeError: If collection creation fails.
    """
```

**Algorithm**:

1. List existing collections via `self.db.list_collection_names()`
2. If `self.collection_name` exists:
   a. Get collection via `self.db.get_collection(self.collection_name)`
   b. Return collection
3. If not exists:
   a. Create collection with `self.db.create_collection()`:
      - `name=self.collection_name`
      - `dimension=dimension`
      - `metric="cosine"` (cosine similarity)
   b. Return new collection

#### 3.3.3 `insert_chunks`

```python
def insert_chunks(self, chunks: list[EmbeddedChunk]) -> list[str]:
    """Insert embedded chunks into the vector store.

    Args:
        chunks: List of chunks with embeddings.

    Returns:
        List of inserted document IDs in same order.

    Raises:
        RuntimeError: If insertion fails.
    """
```

**Algorithm**:

1. Call `self.ensure_collection()` to get collection
2. Prepare documents list:
   - For each chunk, create dict:
     ```python
     {
         "_id": f"chunk_{chunk['chunk_id']}",
         "text": chunk["text"],
         "$vector": chunk["embedding"],
         "metadata": chunk["metadata"]
     }
     ```
3. Insert documents via `collection.insert_many(documents)`
4. Extract and return list of inserted IDs

#### 3.3.4 `search_similar`

```python
def search_similar(
    self,
    query_embedding: list[float],
    top_k: int = 5,
    filter_dict: dict | None = None,
) -> list[dict]:
    """Search for similar chunks using vector similarity.

    Args:
        query_embedding: Query vector to search for.
        top_k: Number of results to return.
        filter_dict: Optional metadata filter (e.g., {"chunk_type": "code_walkthrough"}).

    Returns:
        List of matching documents with similarity scores.
    """
```

**Algorithm**:

1. Call `self.ensure_collection()` to get collection
2. Build find options:
   - `vector=query_embedding`
   - `limit=top_k`
   - `include_similarity=True`
   - If `filter_dict` → `filter=filter_dict`
3. Execute `collection.find({}, options)`
4. Return list of documents with `$similarity` field

#### 3.3.5 `delete_all_chunks`

```python
def delete_all_chunks(self) -> int:
    """Delete all chunks from the collection.

    Returns:
        Number of documents deleted.

    Raises:
        RuntimeError: If deletion fails.
    """
```

**Algorithm**:

1. Get collection via `self.ensure_collection()`
2. Call `collection.delete_many({})` to delete all documents
3. Return delete count

### 3.4 `embedding_pipeline.py`

**Purpose**: Orchestrate embedding generation and vector store insertion.

#### 3.4.1 `run_embedding_pipeline`

```python
def run_embedding_pipeline(
    chunks_json_path: str,
    embedding_status_path: str = "data/embeddings/embedding_status.json",
    batch_size: int = 50,
) -> dict:
    """Run the full embedding and vector store pipeline.

    Reads chunks from chunking pipeline output, generates embeddings using
    MeshAPI, and inserts into AstraDB vector store.

    Args:
        chunks_json_path: Path to chunks.json from chunking pipeline.
        embedding_status_path: Path to write status tracking file.
        batch_size: Number of chunks to embed per batch.

    Returns:
        dict with keys:
            "total_chunks": int — total chunks processed
            "embedded": int — chunks successfully embedded
            "stored": int — chunks stored in vector DB
            "failed": int — chunks that failed
            "errors": list[str] — error messages
            "status_path": str — path to status file

    Raises:
        FileNotFoundError: If chunks_json_path does not exist.
        ValueError: If required environment variables are missing.
    """
```

**Algorithm**:

1. **Validate environment**:
   - Check `MESH_API_KEY` env var is set
   - Check `ASTRA_API_ENDPOINT` and `ASTRA_TOKEN` env vars are set
   - Raise `ValueError` if any missing

2. **Initialize clients**:
   - `chunk_reader.read_chunks(chunks_json_path)` → get list of `ChunkRecord`
   - `EmbeddingClient()` → for generating embeddings
   - `VectorStoreClient()` → for storing in AstraDB

3. **Prepare tracking structures**:
   - Initialize `status_list: list[EmbeddingStatus] = []`
   - Initialize `errors: list[str] = []`

4. **Process in batches**:
   - For `i` from 0 to `len(chunks)` step `batch_size`:
     a. `batch = chunks[i:i+batch_size]`
     b. Extract texts: `texts = [c["text"] for c in batch]`
     c. **Generate embeddings**:
        - Call `embedding_client.embed_batch(texts)`
        - If success → get list of embedding vectors
        - If fail → log error, mark all chunks in batch as failed, continue to next batch
     d. **Prepare embedded chunks**:
        - For each chunk, create `EmbeddedChunk`:
          ```python
          {
              "chunk_id": chunk["chunk_id"],
              "text": chunk["text"],
              "embedding": embedding_vector,
              "metadata": {
                  "chunk_id": chunk["chunk_id"],
                  "start_time": chunk["start_time"],
                  "end_time": chunk["end_time"],
                  "start_seconds": chunk["start_seconds"],
                  "end_seconds": chunk["end_seconds"],
                  "duration_seconds": chunk["duration_seconds"],
                  "topic_hint": chunk["topic_hint"],
                  "chunk_type": chunk["chunk_type"],
                  "source_file": chunk["source_file"],
                  "cue_count": chunk["cue_count"],
              }
          }
          ```
     e. **Insert into vector store**:
        - Call `vector_store_client.insert_chunks(embedded_chunks)`
        - If success → mark all chunks as embedded + stored
        - If fail → log error, mark all as failed

5. **Write status file**:
   - Create output directory if needed
   - Write `embedding_status.json` with status for each chunk

6. **Return summary dict** with counts and error list

#### 3.4.2 `_validate_env`

## 9. Error Handling Strategy

### 9.1 Error Types

| Error Type | Module | Handling |
|------------|--------|----------|
| Missing env vars | `embedding_pipeline` | Raise `ValueError` with list of missing vars |
| File not found | `chunk_reader` | Raise `FileNotFoundError` |
| Invalid JSON | `chunk_reader` | Raise `ValueError` |
| API error (embedding) | `embedding_client` | Raise `RuntimeError` with error details |
| API rate limit | `embedding_client` | Retry with exponential backoff (3 retries) |
| AstraDB connection | `vector_store_client` | Raise `RuntimeError` |
| AstraDB insert fail | `vector_store_client` | Log error, continue with other batches |
| Partial batch failure | `embedding_pipeline` | Track in status file, continue pipeline |

### 9.2 Retry Strategy

| Operation | Retries | Backoff | Conditions |
|-----------|---------|---------|------------|
| MeshAPI embed | 3 | 2s → 4s → 8s | Rate limit, timeout, 5xx errors |
| AstraDB insert | 3 | 1s → 2s → 4s | Timeout, connection error |
| AstraDB query | 2 | 1s → 2s | Timeout only |

## 10. Performance Considerations

### 10.1 Batch Sizes

| Operation | Default Batch Size | Rationale |
|-----------|-------------------|-----------|
| Embedding | 100 | MeshAPI optimal for 100 texts |
| AstraDB insert | 50 | Avoid request size limits |
| Pipeline processing | 100 | Memory/throughput balance |

### 10.2 Memory Management

- Stream chunks from JSON instead of loading all into memory (for large files)
- Process batches sequentially, not in parallel (API rate limits)
- Clear embedding vectors from memory after successful insert

### 10.3 Expected Performance

| Metric | Target |
|--------|--------|
| Embedding throughput | ~500 chunks/minute |
| AstraDB insert throughput | ~1000 chunks/minute |
| Total pipeline (1000 chunks) | ~3 minutes |
